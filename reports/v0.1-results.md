verification only and are not public benchmark claims.

## Expected Interpretation

The regex baseline is intentionally transparent and brittle. It should be used
only as a floor for comparing simple pattern matching against richer context
health evaluation.

The CtxGov adapter no longer mirrors labels. Its default heuristic mode is a
transparent pattern adapter and performs poorly on the original v0.1 cases. Use
doctor mode with a local CtxGov checkout when measuring real CtxGov behavior.

## Error Analysis

Regex baseline false positives:

- `ach-004`: predicted `conflicting_policy`; expected `stale_claim`. The word
  `hosted runtime` triggered a conflict-shaped rule even though the case is a
  stale demo-readiness claim.
- `ach-007`: predicted `hidden_terminal_failure`; expected `stale_claim`. The
  command failure is present, but the labeled target is the stale pass claim.

Regex baseline false negatives:

- `ach-004`: missed `stale_claim` for `live demo is ready`.
- `ach-007`: missed `stale_claim` for `all tests passed`.
- `ach-041`: missed `hidden_terminal_failure` for a blank browser capture.
- `ach-042`: missed `hidden_terminal_failure` for skipped Pages deployment.

The heuristic adapter's low v0.1 score is expected: it is intentionally narrow
and does not read labels. It is useful for exercising the output format, not for
benchmark claims.
