# v0.8 Eval Hardening Results

Date: 2026-06-05
Status: public local evaluation hardening artifact. No public benchmark claim.

## Summary

v0.8 adds public-safe hard negatives, CtxGov self-audit cases, and a blind
reviewer protocol. The purpose is to reduce the credibility risk created by
perfect scaffold metrics in earlier local suites.

## Dataset

| Split | Cases | Labels | Hard negatives | Self-audit positives |
| --- | ---: | ---: | ---: | ---: |
| `eval_hardening_v0.8` | 54 | 54 | 50 | 4 |

Hard-negative categories:

- quoted false examples
- historical release references
- blocked instructions
- fixed terminal failures
- provenance-only legacy namespace mentions
- old license discussion
- claim-boundary quotes
- static demo scope
- package non-claims
- bounded reviewer offers

Self-audit finding types:

- `publication_state_drift`
- `repo_map_drift`
- `version_surface_drift`
- `roadmap_pointer_drift`

## Why This Matters

Earlier local scaffold suites can produce perfect scores for transparent
baselines. v0.8 intentionally includes rows that look risky but should not be
flagged because the surrounding context marks them as quoted, historical,
fixed, blocked, provenance-only, or explicitly negated.

That makes the artifact more reviewer-grade without turning it into a public
benchmark claim.

## Review Path

Use:

```bash
python3 scripts/generate_v08_eval_hardening.py
python3 scripts/check_v08_reviewer_packet.py
python3 -m unittest tests.test_v08_eval_hardening -v
```

The blind reviewer sheet is `review/v08-reviewer-sheet-template.csv`. It
contains cases and reviewer fields only. It intentionally does not expose owner
labels or hidden label columns.

## Baseline Stress Result

The transparent regex baseline is expected to struggle on this split. On the
v0.8 hardening cases it produced 25 false positives on hard negatives and 4
false negatives on self-audit drift cases. That is the intended signal: v0.8 is
designed to expose scaffold brittleness, not preserve a perfect score.
