# Agent Context Health Eval v0.4 Results

Status: public evaluation artifact update. This is not a benchmark claim.

## What v0.4 Adds

- 20 hard-negative cases under `data/v0.4/` that mention releases, failures,
  rollback, schema, memory, and approval without requiring a finding.
- A tightened regex baseline that matches labeled evidence phrases instead of
  isolated hazard vocabulary such as `failed`, `repo exists`, or `may write`.
- Expanded CtxGov doctor adapter mappings for release integrity, Memory X-Ray
  L1, and Task Shard context-control findings.
- A single-label projection layer for the current trace-pattern scorer. It uses
  public `benchmark_family` metadata, not labels, to pick the primary finding
  when CtxGov emits multiple valid findings for one case.

## Comparison To v0.3

v0.3 exposed a real product gap: CtxGov doctor scored 0.3158 precision, 0.1200
recall, and 0.1739 F1 on the v0.2 trace-pattern split because the doctor did
not yet have native release integrity, Memory X-Ray L1, or Task Shard checks.

v0.4 closes that scaffold gap by adding native CtxGov finding types and exact
evidence spans. This confirms the evaluation artifact is useful as a product
coverage harness, but it still does not establish general benchmark validity.

## False Positive / False Negative Analysis

- v0.4 CtxGov doctor adapter: no FP/FN on the public v0.2 trace-pattern split.
- v0.4 regex baseline: no FP on the 20 synthetic hard-negative controls.
- The earlier v0.3 doctor false negatives were concentrated in release
  integrity, Memory X-Ray L1, Task Shard schema conflict, and pending approval
  contradiction cases.
- The earlier v0.3 doctor false positives came from generic release/action
  findings emitted alongside more specific native findings. v0.4 projects those
  multi-finding outputs into the current single-label scorer.

## Claim Boundary

The 1.0000 scores are readiness signals for the scaffold, adapter contract,
doctor coverage, and evidence-span extraction on public trace-pattern data.
They are not public benchmark claims. The data is still regular, partially
synthetic, and not independently adjudicated.

Before claiming a benchmark result, this needs independent reviewer labels,
adjudication, a hidden holdout administered outside the repo, more hard
negatives, and a multi-label scoring protocol for cases where multiple findings
are valid.
