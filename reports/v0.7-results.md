## Offline Adapters

v0.7 includes offline adapters for:

- GitHub PR/release/issues payloads
- CI/terminal log text
- AGENTS/CLAUDE/Cursor-style rules text
- package registry manifests
- local transcripts
- memory trace metadata

The adapters accept local text or payloads and do not fetch remote state.

## Automated Error Analysis

Generated artifacts:

- `reports/v0.7-regex-error-analysis.json`

- `reports/v0.7-ctxgov-doctor-error-analysis.json`

clustered around release-notes cases that contain memory-L1 phrases. This is a
useful local signal about source-family confusion.

## Reproduce

```bash
python3 scripts/generate_v07_trace_suite.py
python3 baselines/regex_baseline.py --cases data/v0.7/trace_shaped_cases.jsonl --output reports/v0.7-regex-baseline-results.jsonl --multi-label
python3 ctxgov_adapter/run_ctxgov.py --cases data/v0.7/trace_shaped_cases.jsonl --output reports/v0.7-ctxgov-doctor-results.jsonl --mode doctor --projection none --ctxgov-root /path/to/ctxgov
python3 scoring/score_multilabel.py --labels data/v0.7/trace_shaped_labels.jsonl --predictions reports/v0.7-ctxgov-doctor-results.jsonl
python3 scoring/error_analysis.py --labels data/v0.7/trace_shaped_labels.jsonl --predictions reports/v0.7-ctxgov-doctor-results.jsonl --hard-negative-labels data/v0.7/trace_shaped_labels.jsonl --output reports/v0.7-ctxgov-doctor-error-analysis.json
```

## Limitations

No public benchmark claim. No security guarantee. No provider/model call. No
network-backed GitHub or package-registry check. No hosted runtime claim. The
suite is synthetic trace-shaped local data designed to exercise schema,
adapters, scoring, and report workflows before externally reviewed traces are
available.
