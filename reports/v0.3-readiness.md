## Critical Interpretation

The v0.2 regex and heuristic scores of 1.0000 are not benchmark claims. The
public trace-pattern data is regular enough that transparent rules can match it
directly. Those runs prove the plumbing: schema, scorer, adapter contract, and
evidence-span metric.

The real CtxGov doctor adapter score is the useful signal:

| Evaluator | Precision | Recall | F1 | Mean evidence token-F1 |
| --- | ---: | ---: | ---: | ---: |
| regex baseline | 1.0000 | 1.0000 | 1.0000 | 0.9143 |
| CtxGov heuristic adapter | 1.0000 | 1.0000 | 1.0000 | 0.5803 |
| CtxGov doctor adapter | 0.3158 | 0.1200 | 0.1739 | 0.2000 |

The doctor result means CtxGov currently catches some existing context-health
families, especially hidden terminal failure patterns, but does not yet have
native coverage for several evaluation families introduced by the companion
artifact.

## Product Facts Exposed

- Release integrity needs native finding types rather than mapping everything
  through generic unsupported or unsafe claims.
- Memory X-Ray L1 needs first-class checks for source coverage, rollback,
  consequence ceilings, and model-state surface.
- Task Shard context control needs explicit import/schema/approval/rollback
  checks.
- Evidence span quality matters. Generic doctor reasons reduce evidence-span
  overlap even when the broad finding type is right.
- Public pattern data is too easy. Future gains should be measured on
  reviewer-adjudicated or withheld cases, not on the current trace-pattern
  split.
