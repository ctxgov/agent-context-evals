# Agent Context Health Eval v0.2 Results

Status: public scaffold results. These numbers validate dataset, scorer,
adapter, and report mechanics. They are not public benchmark claims.

## Dataset

- v0.2 trace-pattern labeled cases: 50
- hidden holdout public case texts: 12
- hidden holdout public labels: no
- benchmark families:
  - `memory_xray_l1_eval`
  - `release_integrity_claim_lint_eval`
  - `session_continuity_hidden_holdout_eval`
  - `task_shard_context_control_eval`

## Scorer Additions

v0.2 adds:

- per finding-type precision, recall, and F1
- token-F1 evidence-span overlap for true positives
- support for Memory X-Ray L1 finding types:
  - `missing_source_coverage`
  - `missing_rollback`
  - `unbounded_consequence`

## Interpretation

The regex and heuristic results are intentionally not benchmark claims. The
data is public and pattern-shaped, so transparent rules can match it directly.
These runs prove that the scorer, labels, evidence-span metric, and adapter
output contract work.

The CtxGov doctor adapter is the important v0.2 gap signal. It runs CtxGov's
local `ctxgov.cli doctor` over materialized case workspaces, but the current
doctor taxonomy does not cover several v0.2 benchmark families. That is why it
finds hidden terminal failures but misses most Memory X-Ray, release-integrity,
and task-shard cases.

## Remaining Gaps

- real trace-derived cases need independent reviewer approval
- hidden holdout labels are withheld but not independently administered
- negative controls need expansion
- CtxGov doctor needs native mappings for release integrity, Memory X-Ray L1,
  and task-shard context control
- no hosted demo, provider/model judge, leaderboard, or security guarantee
